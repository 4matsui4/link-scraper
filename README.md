# link-scraper

JavaScript でレンダリングされる Web ページから、特定ドメインへのリンク要素（`<a href>`）を一括取得するスクレイピングツールです。

## 用途

### 主な利用シーン

Google Tag Manager（GTM）などのタグ管理ツールでクリックトリガーを設定する際、**対象要素の `id` 属性が正しく付与されているか・どの要素が遷移先として設定されているか**を事前に棚卸しするために使います。

たとえば次のような確認作業に適しています。

- LP や複数ページにまたがる CTA ボタンの `id` 一覧を取得したい
- 特定の遷移先 URL に紐づくリンク要素を網羅的に洗い出したい
- デザインツール（Studio.Design / Webflow / Wix など）で生成されたページのように、HTML ソースを直接確認しにくい場合に DOM を確認したい

### 静的スクレイピングとの違い

このツールは **Playwright（ヘッドレス Chromium）** を使って実際にブラウザでページをレンダリングしてから要素を取得します。

`curl` や `requests` などの静的 HTTP 取得では、JavaScript で動的に挿入されるコンテンツを取得できません。コンテンツが API 経由で後から注入されるページには本ツールが必要です。

## セットアップ

```bash
git clone https://github.com/4matsui4/link-scraper
cd link-scraper

npm install
npm run setup        # Chromium を初回インストール（約 100MB）

cp .env.example .env
# .env を編集（後述）
```

## 設定（.env）

```env
# 遷移先として検索したいドメイン（部分一致）
TARGET_DOMAIN=example.com

# スクレイピング対象のページ URL（カンマ区切り）
SOURCE_URLS=https://page1.example.com/,https://page2.example.com/
```

`.env` は `.gitignore` で除外されています。対象 URL をリポジトリにコミットしないよう注意してください。

## 実行

```bash
npm start
```

結果はコンソールに出力されます。最後にタブ区切りのサマリーテーブルが表示されるので、スプレッドシートに貼り付けて利用できます。

## 利用上の注意

### サーバー負荷について

- 本ツールはページを **1 件ずつ順次取得** します。並列実行は行いません。
- 多数のページを連続してスクレイピングする場合は、コードにウェイト処理（`setTimeout`）を追加してリクエスト間隔を空けてください。
- 短時間に同一ドメインへ大量リクエストを送ることはサーバーに負荷をかけるため、**業務上必要な範囲に留めてください。**

### robots.txt の確認

対象サイトの `robots.txt`（例：`https://example.com/robots.txt`）を事前に確認し、クローラーアクセスが禁止されていないことを確認してください。

### 利用規約・法的事項

- **対象サイトの利用規約を必ず確認してください。** スクレイピングを明示的に禁止している場合は利用しないでください。
- **自分が管理・運営するサイト、または正式に調査権限を持つサイトのみ**を対象として使用することを推奨します。
- 取得したデータに個人情報が含まれる可能性がある場合は、個人情報保護法（日本）および GDPR 等の適用法令に従って取り扱ってください。

## 技術仕様

| 項目 | 内容 |
|---|---|
| ブラウザエンジン | Chromium（Playwright 管理） |
| 待機条件 | `networkidle`（通信が落ち着くまで待機）+ 3 秒の追加待機 |
| 取得対象 | `<a href>` 要素 / `onclick` 属性を持つ要素 |
| URL 管理 | `.env` ファイルで外部化（リポジトリに含まない） |

## 依存パッケージ

- [playwright](https://playwright.dev/) — ヘッドレスブラウザ操作
- [dotenv](https://github.com/motdotla/dotenv) — 環境変数の読み込み
